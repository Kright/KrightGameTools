## Plane-based geometric algebra for 2d

Contains generated code for special cases: lines, rotors, points, etc.
For example, the point class contains only two fields and rotor has only two (cos and sin of the half-angle).
So rotation of point by rotor is efficient as usual code in math libraries.

The ideas shared by the pga2d/pga3d modules — the products (geometric, dot, meet, join, sandwich),
bulk and weight, duality, exp/log, the narrowest-result-type rule and immutability — are described
once in [pga-concepts.md](../pga-concepts.md).

Full operation/result-type reference: [operations.md](operations.md).

The code is generated by the [pgaNdCodeGen](../pgaNdCodeGen/README.md) module and mirrors
[pga3d](../pga3d/README.md) with one naming change: a grade-1 element is called a line
(in 2d a hyperplane is a line), so Pga2dLine is a sibling of Pga3dPlane.

### Line:

* [**Pga2dLine**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dLine.scala): line, 3 fields. Sibling of Pga3dPlane
* [**Pga2dLineIdeal**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dLineIdeal.scala): line passing through the center of coordinates, 2 fields. Dual to Pga2dVector

```scala
val line = Pga2dLine(a, b, c) // ax + by + c = 0
val idealLine = Pga2dLineIdeal(a, b) // ax + by = 0
val point = Pga2dPoint(x, y)

// Projecting a point onto a line
val projectedPoint = point.projectOntoLine(line)

// Mirroring by a line
val mirroredPoint = line.sandwich(point)
```

### Point:

In 2d there is no bivector family as in pga3d: the grade-2 elements are the points themselves.
The exponent of a grade-2 element is a motor, and the logarithm of a motor is a grade-2 element.

* [**Pga2dPoint**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dPoint.scala): Point on the plane. Stored as dual representation with human-friendly fields x, y and fixed w=1.
* [**Pga2dPointCenter**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dPointCenter.scala): singleton object, center of coordinates.
* [**Pga2dVector**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dVector.scala): difference between points. Consist of x, y and fixed w=0. Pga2dTranslator moves points, but not vectors.
* [**Pga2dProjectivePoint**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dProjectivePoint.scala): general case with three homogeneous coordinates (x, y, w). Could be mapped to point via (x/w, y/w) when w != 0.

```scala
// Creating a point
val point1 = Pga2dPoint(x, y)
val point2 = Pga2dPoint(x2, y2)

// difference between points is a vector
val vector = point1 - point2

// sum of vector and point is a point
val point3 = vector * 2 + point1

// distance between points
val distance = point1.distanceTo(point2)

// creating point as intersection of two lines
val point4 = lineA meet lineB
val point4_ = lineA ^ lineB

// Creating a line from two points
val lineThroughPoints = point1 join point2
val lineThroughPoints_ = point1 v point2
```

### Movement

* [**Pga2dRotor**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dRotor.scala): represents rotation, 2 fields (scalar, xy) - cos and sin of the half-angle. The 2d analog of Pga3dRotor, it is the exponent of a rotation around the center
* [**Pga2dTranslator**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dTranslator.scala): represents linear movement, 2 fields (wx, wy). It is the exponent of Pga2dVector
* [**Pga2dMotor**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dMotor.scala): combination of rotation and linear movement, 4 fields (scalar, wx, wy, xy) - the even subalgebra. Exponent of Pga2dProjectivePoint

To move everything with these classes, you need to call `motor.sandwich(obj)`

```scala
// Creating a rotor for rotation between two vectors
val from = Pga2dVector(1, 2)
val to = Pga2dVector(3, 4)
val rotor = Pga2dRotor.rotation(from, to)

// Restoring a rotor from a rotated orthonormal basis (columns of a 2x2 rotation matrix)
val rotor2 = Pga2dRotor.restore(axisX, axisY)

// Applying rotation to anything using the sandwich product (could rotate point, line, rotor, etc.)
val rotatedPoint = rotor.sandwich(point)

// Inverse of rotation
val inverseRotation = rotor.reverse

// Apply inverse of rotation in rotor
val restoredPoint = rotor.reverseSandwich(point)
val restoredPoint2 = rotor.reverse.sandwich(point)

// Creating a translator to add a vector
val vector = Pga2dVector(1, 2)
val translator = Pga2dTranslator.addVector(vector)

// Applying translation to a point
val translatedPoint = translator.sandwich(point)
// This is equivalent to:
val translatedPoint2 = point + vector

// Creating a motor by combining a translator and a rotor
val motor = translator.geometric(rotor)

// Applying a motor to a point (combined rotation and translation)
val transformedPoint = motor.sandwich(point)

// Computing the logarithm of a motor (a grade-2 element, i.e. a projective point)
val logarithm = motor.log()

// Computing the exponential of a grade-2 element (results back in a motor)
val motor2 = logarithm.exp()

// Interpolating between two rotors/motors: slerp() follows the exact geodesic (constant
// angular velocity), nlerp() is a cheaper renormalized-lerp approximation of it.
// (motor.renormalized is a uniform scale in 2d, unlike the 3d motor case.)
val halfway = rotor.slerp(rotor2, 0.5)
val approxHalfway = rotor.nlerp(rotor2, 0.5)
```

### Other classes:

* [**Pga2dMultivector**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dMultivector.scala): class with all 8 fields for a general case
* [**Pga2dPseudoScalar**](shared/src/main/scala/me/kright/gametools/pga2d/Pga2dPseudoScalar.scala): class with one field (xyw). Library has no scalar class and uses just Double instead.

```scala
// Converting specialized classes to multivector
val multivectorFromPoint = point.toMultivector
val multivectorFromRotor = rotor.toMultivector

// Creating a pseudoscalar
val pseudoscalar = Pga2dPseudoScalar(value)
```
